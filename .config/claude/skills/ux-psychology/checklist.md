# UX Psychology Implementation Checklist

Use this checklist to review your UI against psychological principles.

## Performance

- [ ] Is initial display under 400ms? (Doherty Threshold)
- [ ] Is there skeleton UI for slow loads?
- [ ] Is progress shown during loading? (Labor Illusion)

## Information Architecture

- [ ] Is cognitive load appropriate?
- [ ] Is visual hierarchy clear?
- [ ] Does important information stand out? (Selective Attention)
- [ ] Are critical messages outside banner formats? (Banner Blindness)

## Behavioral Guidance

- [ ] Is the CTA clear? (Visual Anchor)
- [ ] Are defaults appropriate? (Default Bias)
- [ ] Is feedback immediate?
- [ ] Are prices anchored properly? (Anchor Effect)
- [ ] Is information framed positively? (Framing Effect)

## Engagement

- [ ] Are progress indicators used effectively? (Goal Gradient)
- [ ] Are incomplete tasks highlighted? (Zeigarnik Effect)
- [ ] Is gamification appropriate for the context?

## Value Communication

- [ ] Is scarcity shown when legitimate? (Scarcity Effect)
- [ ] Is personalization offered? (Endowment Effect)
- [ ] Are cancellation consequences clear? (Loss Aversion)

## Trust

- [ ] Is there social proof? (Reviews, counts, logos)
- [ ] Are error states helpful and graceful?
- [ ] Are completion states delightful? (Peak-End Rule)

## User Respect

- [ ] Not being pushy? (Reactance)
- [ ] Providing choice freedom?
- [ ] Avoiding dark patterns?
- [ ] Using intentional friction only for protection?
- [ ] Minimizing decision fatigue?

## Navigation & Layout

- [ ] Following familiar patterns? (Familiarity Bias)
- [ ] Important items at first/last positions? (Serial Position Effect)
- [ ] Progressive disclosure where appropriate?
- [ ] Navigation limited to 5 items or fewer?
